## 1. 진단 요약

### 1.1 발견된 핵심 이슈

| # | 이슈 | 심각도 | 영향 |
|---|------|--------|------|
| 1 | sitemap에 색인 가능한 페이지 6개뿐 (`/reviews` noindex 후 실질 5개) | 🔴 매우 높음 | 색인량 부족 |
| 2 | `books/[isbn]` 책 상세 페이지가 sitemap 미포함 | 🔴 매우 높음 | 수많은 페이지가 색인 누락 |
| 3 | 콘텐츠가 외부 API(Naver, Data4Library) 의존적 → 원본성 부족 | 🔴 매우 높음 | "Low value" 직접 사유 |
| 4 | 자체 작성 텍스트 콘텐츠 부족 (블로그/매거진 섹션 없음) | 🔴 매우 높음 | 원본 가치 입증 불가 |
| 5 | about 페이지 콘텐츠 얕음 (151줄, 실제 텍스트는 더 적음) | 🟡 중간 | 사이트 신뢰도 약함 |
| 6 | privacy 페이지 단일 (terms/cookie policy 분리 안 됨) | 🟡 중간 | 표준 양식 미달 |

### 1.2 AdSense 가이드 기준 부합도 (참고 글 기반)

| 기준 | 목표 | 현재 | 상태 |
|------|------|------|------|
| 고품질 원본 글 | 10~20개 | 0개 (블로그 없음) | ❌ |
| 글당 글자 수 | 1,500자 이상 | N/A | ❌ |
| About 페이지 | 충실한 소개 | 얕음 | ⚠️ |
| Privacy Policy | 표준 양식 | 단순 | ⚠️ |
| Contact 페이지 | 명확한 연락처 | 존재 | ✅ |
| 모바일 반응형 | 필수 | 확인 필요 | ⚠️ |
| 저작권 이슈 없는 이미지 | 필수 | 확인 필요 | ⚠️ |

## 2. 작업 계획

전체 작업을 3단계로 나누어 진행한다. **1단계만 완료해도 재심사 통과 가능성이 크게 높아진다.**

### 2.1 1단계 — 즉시 실행 (1~2주, AdSense 통과 핵심)

#### A. 블로그 / 인사이트 섹션 신설 ⭐ 최우선

**목표**: 1,500자 이상 원본 글 10개 이상 발행

**구조 설계**
- 경로: `apps/web/src/pages/insights/[slug].astro`
- 콘텐츠 형식: 마크다운 (Astro Content Collections 활용)
- 콘텐츠 위치: `apps/web/src/content/insights/*.md`
- 인덱스 페이지: `apps/web/src/pages/insights/index.astro`
- sitemap 자동 포함 (Astro sitemap integration 또는 수동 갱신)

**작성할 글 주제 후보 (10개)**
1. 2026년 도서관 대출 TOP 100 분석 — 자체 데이터 기반
2. 지역별 인기 장르 비교 (서울 vs 부산 vs 광역시) — 자체 데이터
3. 베스트셀러가 도서관에 들어오는 평균 기간 분석
4. 전자책 vs 종이책 대출 트렌드 (분기별 변화)
5. "도서관 인사이트" 사이트를 만든 이유 — 운영자 글
6. 공공도서관 200곳 데이터로 본 독서 인구 변화
7. 학술서 vs 대중서 대출 비율 — 도서관 유형별
8. 신간이 베스트셀러가 되기까지 평균 일수
9. 도서관 자료 검색을 200% 활용하는 팁
10. 2025 연말 결산 — 가장 많이 빌린 책, 가장 빨리 빌려간 책

**작성 가이드라인 (참고 글 기준)**
- 글 한 편당 1,500~2,000자
- 100% 자체 작성, 표절 금지
- 문법 오류 없이 깔끔하게
- "광고 클릭해 주세요" 같은 유도 문구 절대 금지
- 이미지는 Unsplash / Pixabay / Pexels 등 무료 출처만 사용
- 이미지 ALT 태그에 키워드 포함 (SEO)
- 자체 데이터로 만든 차트/표 적극 활용 (원본성 입증에 효과적)

**기술 작업 체크리스트**
- [x] `astro:content` Content Collections 스키마 정의 (`src/content/config.ts`)
- [x] `[slug].astro` 라우트 + 마크다운 렌더링 (`src/pages/insights/[slug].astro`)
- [x] 인사이트 인덱스 페이지 — 글 목록, 카테고리 뱃지, 발행일 정렬 (`src/pages/insights/index.astro`)
- [x] 발행 글의 sitemap.xml 수동 갱신 (6개 → 19개 URL)
- [x] OpenGraph / Twitter 메타 태그 (슬러그 페이지·인덱스 모두 적용)
- [x] 카테고리·발행일 표시 (읽는 시간은 미구현 — 추후 추가 가능)
- [ ] 관련 글 추천 영역 (현재 "목록으로 돌아가기" 링크만 — 추후 개선)
- [ ] 모바일 가독성 실기기 확인

**글 작성 현황 (10/10 완료)**
- [x] 2026년 도서관 대출 TOP 100 분석
- [x] 지역별 인기 장르 비교 (서울 vs 부산 vs 광주)
- [x] 베스트셀러가 도서관에 들어오기까지 평균 며칠?
- [x] 전자책 vs 종이책 대출 트렌드 5년 변화
- [x] LibraryInsights를 만든 이유
- [x] 공공도서관 200곳 데이터로 본 독서 인구 변화
- [x] 학술서 vs 대중서 도서관 유형별 대출 비율
- [x] 신간이 도서관 베스트셀러가 되기까지
- [x] 도서관 검색 200% 활용하는 7가지 팁
- [x] 2025 연말 결산

**이미지 수정** (2026-04-27)
- [x] 3·6·9번째 글 깨진 이미지 교체 (6·9번 URL 중복 포함)

#### B. about 페이지 대폭 확장

**기술 작업 체크리스트**
- [x] 텍스트 1,500자 이상으로 확장 (운영 철학·사용법·로드맵 섹션 추가)
- [ ] 시각 자료 (사이트 스크린샷, 데이터 흐름 다이어그램) 추가 — 추후 개선
- [x] 구조화 데이터 (`Organization` schema.org) 삽입

#### C. Privacy Policy / Terms / Cookie Policy 분리 강화

**작업 항목**
- [x] `/privacy` — 9개 → 13개 섹션으로 확장 (수탁업체 표, 권익침해 구제 방법 포함)
- [x] `/terms` 신규 작성 (11조, 광고 면책 조항 포함)
- [x] `/cookie-policy` 신규 작성 (AdSense·Analytics 쿠키 상세 안내)
- [x] 푸터에서 세 페이지 모두 링크 (기존 버튼 → 페이지 링크로 교체)

### 2.2 2단계 — 책 상세 페이지 강화 (1주, 1단계 후 진행)

#### A. 책 상세 페이지에 자체 인사이트 추가

**현재 문제**: 외부 API 메타데이터를 그대로 표시 → 원본성 부족

**추가할 자체 생성 콘텐츠**
- 이 책의 도서관 보유 분포 분석 (지역별 차트)
- 비슷한 책 자동 추천 (분류 코드 + 자체 알고리즘, 추천 이유 명시)
- 지역별 대출 가능성 점수
- 이 책의 인기도 시계열 (월별 대출 추이가 가능하다면)
- 같은 저자의 다른 책 그리드

**기술 작업 체크리스트**
- [ ] 추천 알고리즘 설계 (분류 + 출판사 + 발행일 가중치)
- [ ] 대출 가능성 점수 계산 로직
- [ ] 차트 컴포넌트 (Recharts 또는 Chart.js)
- [ ] 추천 이유 자연어 생성 (간단한 템플릿)
- [ ] 책별 페이지에 1,000자 이상의 자체 분석 텍스트 자동 노출

#### B. 동적 sitemap 생성

**현재**: `apps/web/public/sitemap.xml` 정적 (6개 URL)

**작업 항목**
- [ ] sitemap.xml 동적 생성 엔드포인트 (`/sitemap.xml.ts`) 또는 빌드 시 생성
- [ ] DB에 저장된 책 ISBN 목록을 자동 포함
- [ ] insights 글 자동 포함
- [ ] 갱신일(`lastmod`) 정확히 반영
- [ ] 50,000 URL 초과 시 sitemap index 분할 고려
- [ ] Search Console에 재제출

**주의**: 콘텐츠가 빈약한 책 페이지를 sitemap에 등록하면 오히려 역효과. **2.2-A 완료 후** sitemap 등록 진행.

### 2.3 3단계 — 장기 (1단계 통과 후 천천히)

- 사용자 리뷰 시스템 (별점 + 코멘트) — 사용자 생성 콘텐츠는 100% 원본
- 도서관별 상세 페이지 `/libraries/[code]` — 보유 도서, 위치, 운영시간, 인기 책
- 검색 결과 페이지 SEO 최적화 (현재 noindex라면 일부만 풀기)
- 책 상세 페이지에 사용자 리뷰 임베드
- 카테고리별 큐레이션 페이지 (`/genre/소설`, `/genre/자기계발` 등)

## 3. AdSense 재심사 체크리스트

(참고 글 기준 + 본 사이트 추가 항목)

### 3.1 콘텐츠
- [ ] 1,500자 이상 원본 글 10개 이상 발행
- [ ] 표절 검사 통과 (모든 글)
- [ ] 문법/오타 점검 완료
- [ ] 광고 유도 문구(`광고 클릭` 등) 전체 사이트에서 제거 확인
- [ ] 무료 출처(Unsplash/Pixabay/Pexels) 또는 자체 제작 이미지만 사용
- [ ] 모든 이미지에 ALT 태그 부여

### 3.2 필수 페이지
- [ ] About — 충실한 소개
- [ ] Privacy Policy — 한국 표준 양식
- [ ] Terms of Service — 신규 작성
- [ ] Cookie Policy — AdSense 필수
- [ ] Contact — 명확한 연락 수단
- [ ] 모든 페이지가 푸터에 링크되어 있음

### 3.3 기술
- [ ] 모바일 반응형 동작 확인 (Lighthouse Mobile 점수 80+)
- [ ] 페이지 로딩 속도 (LCP 2.5s 이내)
- [ ] 모든 깨진 링크 수정
- [ ] 404 페이지 커스터마이즈
- [ ] sitemap.xml에 모든 콘텐츠 페이지 포함
- [ ] robots.txt 점검 (필요한 페이지는 Allow)
- [ ] Search Console에 색인 요청
- [ ] Search Console에서 색인 페이지 수가 충분히 늘었는지 확인 (최소 30+)

### 3.4 정책 준수
- [ ] 저작권 침해 가능성 있는 콘텐츠 없음
- [ ] 성인/도박/폭력 등 정책 위반 콘텐츠 없음
- [ ] AdSense 정책 페이지 재숙독

### 3.5 재심사 직전
- [ ] 변경된 모든 페이지가 배포 환경에 반영되었는지 확인
- [ ] Search Console에서 색인 진행 상황 점검 (1주 정도 기다린 후 신청)
- [ ] AdSense 사이트 추가 → 심사 신청
